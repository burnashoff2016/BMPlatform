# Временное отключение аутентификации

## Описание проблемы
Возникли проблемы с системой аутентификации, и стандартные методы входа (логин/пароль) не работают. Пользователи не могут получить доступ к защищенным маршрутам системы.

## Решение
Для временного решения проблемы был реализован способ обхода аутентификации, позволяющий получить доступ ко всем ресурсам системы без ввода учетных данных.

## Инструкции по отключению аутентификации

### Вариант 1: Быстрый обход (рекомендуется для разработки)

1. Откройте файл `backend/main.py`
2. Найдите все строки, содержащие `Depends(get_current_user)`
3. Замените их на `Depends(get_current_user_disabled)` (эта функция уже добавлена в систему)

Пример:
Было:
```python
current_user: User = Depends(get_current_user)
```

Стало:
```python
current_user: User = Depends(get_current_user_disabled)
```

### Вариант 2: Использование фиктивной функции пользователя

В файл `backend/main.py` была добавлена функция `get_current_user_disabled`, которая создает и возвращает тестового пользователя без проверки учетных данных.

### Важные замечания

⚠️ **ПРЕДУПРЕЖДЕНИЕ**: Не используйте этот метод в продакшене без дополнительных мер безопасности!

1. После устранения проблемы с аутентификацией обязательно восстановите нормальный процесс проверки учетных данных.
2. Этот метод предназначен только для временного использования на этапе разработки или тестирования.
3. Все данные, созданные в системе при отключенной аутентификации, будут иметь владельца "anonymous" или "mock_user".

### Как снова включить аутентификацию

1. Верните обратно все зависимости `Depends(get_current_user)` вместо `Depends(get_current_user_disabled)`
2. Удалите или закомментируйте функцию `get_current_user_disabled` из файла `main.py`
3. Убедитесь, что модуль `auth` работает корректно
4. Перезапустите сервер

## Сборка проекта

После внесения изменений соберите проект заново:
```bash
cd backend
pip install -r requirements.txt
python -m uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

Для фронтенд части убедитесь, что установлена переменная среды:
```bash
VITE_API_BASE_URL=http://89.23.97.230/api
```

Затем выполните стандартную сборку фронтенда.
